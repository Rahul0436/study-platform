1st application of java

Here you can practice

    Syntax to compile java application

    • javac filename.java

    Syntax to execute the java application

    • java classname

    Note :

    • javac is the compiler name of java .
    • javac is responsible to compile the java application .
    • jvm is responsible to execute the java application .
    • In a program execution starts from main() method .
    • At the time of compilation javac compiler does not checkc the main() method existance , at runtime jvm first check the main() method existance , if main() method is not available then we will get a run time error " main method not found " .

Rules to write a java program

  • Write a program
  • Save it .
  • Compile it .
  • Execute it .
  • Get the output .

Why save the path of javac compiler ?

  • To access javac compiler any where in our system then we must set the class path of javac compiler.

How to set the path of javac compiler ?

  • There are two different way to set the path of javac compiler.
    • Temporary
    • Permanent
    • Temporary
      • Step 1 : Copy the path of javac compiler(c:/program files/java/jdk/bin)
      • Step 2 : Open command prompt and type here ( set path = ; c:/program files/java/jdk/bin ;)

    • Permanent
      • Step 1 : Copy the path of javac compiler(c:/program files/java/jdk/bin)
      • Step 2 : Go to this pc (right click )
      • Step 3 : Go to properties
      • Step 4 : Advance System setting
      • Step 5 : Go to environment variables
      • Step 6 : Click on path (system variable)
      • Step 7 : Click on edit
      • Step 8 : Click on new
      • Step 9 : Paste the path of javac compiler(c:/program files /java/jdk/bin)

Now let's try to understand one by one what are the element present in 1st application of java

Class

  • Generally ,Class is the collection of homogenious object / similar type of objects .
  • Programatically class is nothing but a keyword which responsible to generate any use defined class .
  • From one class we can create n number of object getting together to create a class .

    • Syntax to create an usedefined class

    • Class Class_name {
    •        //code
    • }

Rules and regulation for naming userdefined class

  • Number are not allowed
  • Number are allowed after string
  • Space is not allowed
  • Underscore is allowed
  • Symbols are not allowed
  • Only dollar is allowed

Naming Convention

  • Class : Each word first letter must be in uppercase

    Example :

    • String
    • System
    • BufferReader
    • InputStreamReader
  • Method : Except fist word next each word first letter should be uppercase

    Example :

    • toString
    • parseInt
    • parseDouble
    • toBinaryString
  • Package : Each letter should lowercase

    Example :

    • java.lang
    • java.util
    • java.io
    • java.awt
    • java.net
  • Constant : Each letter must be uppercase

    Example :

    • MAX_VALUE
    • MIN_VALUE
    • SIZE
    • MAX_PRIORITY
    • MIN_PRIORITY
  • Variable : Except fist word next each word first letter should be uppercase

    Example :

    • toString
    • parseInt
    • parseDouble
    • toBinaryString

Naming Convention

  • A java application contain n number of class
  • If a java application contain n number of class then we can save the file name by any name .
  • After compilation of java application .class file is generated .
  • If a java application contain n number of classes then that class will be executed which contain main method .
  • If a java application contain n number of class then only one class should be declared as a public because of java syntax restriction

      Restriction :

    • if a class is declare as a public that class name and file name must be matched .

    • Native

Member of class

    There are 4 type of members present within the class

  • Data memeber / Variable
  • Method member / Funcition
  • Constructor
  • Block

      Javap :

    • Javap is a tool or command responsible to display member of predefined or userdefined class .

      How to check javap :

    • Compile : javac filename.java
    • Execute : javap classname

Calling procedure

    There are four three different way we can call the member of class

  • Directly
  • By class_name
  • By object_name

      Syntax to create object of class:

    • class_name object_name = new class_name()

      Example :

    • Test obj = new Test()
  • Static can be call directly , by classname and by object name within the same class . But outside the class static member can not be call directly that is call by classname and by objectname .
  • Where as non static member only call by object name within the same class or outside the class .
  • Better for a programmer to call the static member by classname because of avoid memory wastage .
  • When we call the member of a class by class name it does not occupy any memory but when we call the member of a class by object name it occupy space in memory .

Why main method must be declared as a public ?

  • By the syntax of java if a method call from the outside package that method must be declare as a public .
  • Though caller jvm call the main method from the outside package that's why main method must be declared as public .

Why main method static in nature ?

  • The caller jvm call the main method from the outside package without allocate any memory that is by class name .

Why main method return type is void ?

  • When caller jvm call the main method from the outside package , main method does not return any value to the jvm , so main method return type is always void .

What is String[] args ?

  • Here args is nothing but array variable name .
  • We can also change this name but can not change the array type .
  • That is main() method always contains string of array as arguement .

Why System.out.println() ?

  • "println()" is a predefined nonstatic method present in java.io.printstream class .
  • "out" is a predefined object of printstream class but present in java.lang.System class .
  • "System" is a predefined class present in default package in java that is java.lang package .
  • If we involk "println()" method then we write "out.println()" .
  • If we involk "out" object then we must write "System.out" .
  • Combining the above two we get "System.out.println()" .
