Data Types in java

What is data types ?

  • Data types is a keyword or a class which is used to create a variable or object memory to store single value or multiple value as one group .

Why data types ?

  • Data types defines the type of variable (Intger, floating point ,character , boolean , string type ) .
  • Data type also defines size of the variable(1 byte, 2 bytes , 4 bytes , 8 bytes).
  • Data type defines the type of value(int, float , double , char , string ...) and range of the value .

Types of Data types

    Java supports two types of data types

  • Primitive Data types
  • Referenced Data types

Primitve Data types

  • Primitive Data types is used to create a variable for storing single mathematical values .

Referenced Data types

  • Referenced data types is used to create an object for storing multiple value as one group .

Size , range and default value of the data type

Why do we have 8 primitive data type and 8 referenced data type ?

For primitive data type

  • Based on the type of the value and the range of the value java supports 8 primitive data type .
  • The data type int , double , char and boolean are the data type based on type of the value .
  • The data type byte ,short,float and long are the data type based ont the range of the value .
  • For storing lasser range value we must use byte ,short and float and for storing higher range value we must use long data type .

For refferenced data type

  • For storing multiple value as one group we must use refferenced data type .
  • By using primitive data type we are able to store single value at a time . but in a project we must store multiple value at a time , in this case we must use refferenced data type .
  • For storing multiple value of same type we must use array .
  • For storing multiple value of different type we must use class .

What is literal ?

  • Any constant value is called literal .
  • Example : 10, 10.0, 'a',"r", true, null , etc... are literal .

Why literal or what is the use of literal ?

  • A literal is used for :
    • Initializing the variable ( storing value in variable ) .
    • Performing calculation by using operators .
    • Performing validation (checking whether value is currect or not ) .

Types of literal

    Java supports seven types of literal , those are

  • Integer
  • Floating point
  • Character
  • Conditional
  • Class
  • String
  • Null

Integer data type & literals

  • To represent integer type variables and literals we have four data types byte, short , int , long .
  • The data type int is the default data type for integer literal .
  • It means if we type a integer number 10 then compiler and jvm consider it as int type literal(value) .
  • For example : boolean b = 10; // compile time error : int can not be converted to boolean
  • To represent a integer number as byte or short or long type we must write some extra code in three ways
    • By using variable assignment
    • By using cast operator
    • By using suffix character

  • By using variable assignment
  • int i1 = 5;
  • byte b1 = 10;
  • short s1 = 20;
  • long l1 = 30;

  • By using cast operator
  • System.out.println(10);
  • System.out.println((byte)20);
  • System.out.println((short)30);
  • System.out.println((long)40);

  • By using suffix character
  • long l2= 10;
    • or
  • long l3= 10L;
    • or
  • long l4= 10l;
  • int i3 = 10L; error: incompatible types: possible lossy conversion from long to int
  • byte b3 = 10B; or byte b4 = 10b; error: ';' expected
  • short s3 = 10S; or short s4 = 10s; error: ';' expected
  • We can not use suffix character for byte and short . we can represent a integer has byte or short type only in two ways.
  • Assignment operator
  • Cast operator
  • We can not assign any range integer to byte and short , it will show compile time error .
  • The assigned integer must be in the byte range (-128 to 127) and must be short in the short range (-32768 to 32767) .
  • for example :
    • byte b1 = 15;
    • short s = 50;
    • byte b2 = 500; error: incompatible types: possible lossy conversion from int to byte
    • short s2 = 50000; error: incompatible types: possible lossy conversion from int to short
  • Large integer number are also consider as int type .
  • If this large integer number not in the range of int (-2147483648 to 2147483647 ) compiler through error " integer number to large"
  • Even we assign large number to long type variable or even if we use long type cast operator , we will get compile time error "integer number too large".
  • To solve this compile time error we must use suffix "L" or "l" .
    • Storing mobile number , debit card , credit card , account number other number we must use long variable and the number must be suffix with "L " or "l" .

What is the longest integer data type in Java ?

A long data type takes up 64 bits of memory, holds the most integer values, and can be used for very large numbers, ranging from -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807 .

What is the data type of integer in Java?

An integer type stores an integer number with no fractional or decimal places. Java has four integer types – byte, short, int, and long

What is the purpose of integers in Java ?

An integer in Java is a memory location that can hold an integer, a positive or negative non-decimal number. It is denoted by the keyword, 'int'. .

Is integer a keyword in Java ?

The int keyword is used to declare a variable that can hold a 32-bit signed two's complement integer. This keyword is also used to declare that a method returns a value of the primitive type int

Floating point data type and literals

  • To represent floating point numbers java supports two data type , that is float and double .
  • Double is the default data type . it means if we type a floating point number compiler and jvm reads this number as double .
  • To represent a floating point number as float type we must write extra code in two ways .
    • With cast operator
    • With suffix character

      • for example :

    • float f1 = 10.0 ; error:incompatible types: possible lossy conversion from double to float
    • float f2 = 10.0f;(by using suffix character)
    • float f3 = (float)10.0;(by using cast operator )
    • double d = 10.0;

    • We can not assign floating point number directly to float variable , we will get compile time error .

Character data type and literals

  • Any single letter or digit or special character placed inside single quotes is called character .
  • Every character by default is treated as char type to store a single character we must create char variable .

      • For example :

    • char ch1 = 'a';
    • char ch2 = '1';
    • char ch3 = '@';
    • char ch4 = ' ';
  • In a c language char data type occupies 1 byte , because it supports only ASCII character set whose range is 0 to 255 .
  • In java char data type takes 2 bytes because it supports unicode character set whose range is 0 to 65535 .
  • Java supports unicode character set for developing internationalization application(I18N).
  • internationalization application means it supports display content in the current country native language in which this application is run .

Boolean data type and literals

  • To represent conditional or boolean value 'true' or 'false' in java we have a data type boolean .
  • To store 'true' and 'false' we must create boolean type variable as shown below
    • boolean b1 = true;
    • boolean b2 = false;
  • In java , we can not use '1' as true and '0' as false we will get compile time error .
    • boolean b3 = 1; error: incompatible types: int cannot be converted to boolean
    • boolean b4 = 0;error: incompatible types: int cannot be converted to boolean
  • We can not use '0' or '1' as condition value .
  • The range of boolean data type is true , false and size is not fixed , inside jvm it may occupies 1 bit or 1 byte .

String data type and literals

  • A sequence of characters placed inside double quotes is called string or string literals .

      For example :

    • "ram" is a litera .
  • To represent string for creating a variable to store string literal we have a predefined class called String .

      For example :

    • String s1 = "ram";
  • String is not a keyword , it is predefined class available in java.lang package in jdk software . it is a referenced data type .
  • For storing a name , course , dept , any alphanumeric no like pan , engine no , bike no we must use string data type variable .
    • String name = "Ram" ;
    • String course = "Sita";
    • String bikeName = "Bajaj";
    • String engNum = "ABCD123pq";
    • String bikeNum = "TS09BJ999";
  • Inside double quotes " " we can place 0 to n number of character .
  • Empty string literal is allowed .
    • String s1 = "";
    • String s2 = " ";
  • The range of string no limit we can store any no.of character in double quotes.
  • The default value of string is null .

Class Data type and literals

  • After compilation for every class we will get .class file .
  • This .class file name it means classname .class is a literals in java program .
  • For representing .class file name and for storing .class file name we must create a variable by using the predefined class called Class . it is available in java.lang package in jdk software , below program shows .class file name in java program .

      For example :

    • Class c1 = String.class;
    • Class c2 = Test09.class;
  • Every .class file name is a class type literal.
  • To type class name .class , with this class name we must have .class file in the present working directory , else we will get compile time error "can not find symbol".

Null literals

  • Null is a literal of any refferenced type . it means it does not have any specific type .
  • Null is any refferenced type literal means it may be of array /class/interface /enum /anotation type .
  • Null is used for Initializing any refferenced data type (class / array) type variable .
  • Null does not have any specific type .
  • We must assigned type to the null by using two ways
    • By using assignment operator
    • By using cast operator

    • For example :

    • String s = null;
    • Integer i = null;
    • Student s = null;
    • int[] ia = null;
    • Employee[] e = null;
